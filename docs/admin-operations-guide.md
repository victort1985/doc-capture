# Doc Capture — эксплуатация и настройка админ-панели

## 1. Вход

Открыть в браузере адрес сервера (`http://localhost:3000` на самом
сервере или `http://<IP-сервера>:3000` с другого устройства в сети) и
войти под учётной записью администратора.

Разделы в левом меню: **Users** (пользователи), **Storage connections**
(подключения к хранилищам), **Storage routing** (привязка хранилища к
пользователю), **Naming templates** (шаблоны имён файлов), **File log**
(журнал загруженных файлов).

## 2. Пользователи (Users)

- **+ New user** — создать пользователя: имя, пароль (минимум 6
  символов), роль (`admin` — доступ к админ-панели; `user` — только
  вход в мобильное приложение и загрузка файлов), язык по умолчанию
  для мобильного приложения этого пользователя (`he`/`en`/`ru`).
- **Disable/Enable** — временно заблокировать вход без удаления
  учётной записи.
- **Delete** — удалить безвозвратно.

Каждому пользователю **обязательно** настроить привязку хранилища
(раздел 5) — иначе загрузка файлов из мобильного приложения будет
завершаться ошибкой "No storage connection configured".

## 3. Шаблоны имён файлов (Naming templates)

Шаблон определяет, как будет называться файл после обработки на
сервере. Доступные плейсхолдеры:

| Плейсхолдер  | Значение                                  | Пример          |
|--------------|--------------------------------------------|-----------------|
| `{date}`     | Дата загрузки                             | `2026-06-17`    |
| `{time}`     | Время загрузки                            | `14-32-09`      |
| `{place}`    | Поле "Место", введённое в приложении      | `TelAviv`       |
| `{username}` | Имя пользователя, загрузившего файл       | `admin`         |
| `{docType}`  | `document` или `photo`                    | `document`      |
| `{counter}`  | Порядковый номер файла в пакете (с 001)   | `001`           |
| `{uuid}`     | Случайный уникальный идентификатор        | `a1b2c3d4-...`  |

Пример шаблона из коробки: `{date}_{place}_{docType}_{counter}` →
`2026-06-17_TelAviv_document_001.pdf`. Поле **Applies to** ограничивает
шаблон только документами, только фото или обоими типами.

> Поведение шаблона подкаталога (куда внутри хранилища кладётся файл)
> настраивается отдельно — см. раздел 5, "subfolder pattern" — там те
> же плейсхолдеры.

## 4. Подключения к хранилищам (Storage connections)

Здесь создаются сами "точки назначения" для файлов: локальная папка,
FTP-сервер или Synology NAS. Привязка конкретного подключения к
конкретному пользователю — отдельный шаг (раздел 5).

### 4.1 Локальное хранилище (Local filesystem)

- **Type**: Local filesystem
- **Base path**: путь на диске сервера, например `D:\DocCaptureFiles`
  (Windows) или `/data/doc-capture-files` (Linux). Папка будет создана
  автоматически при первой записи, если её нет — но диск должен
  существовать и быть доступен на запись пользователю, под которым
  работает сервер.

### 4.2 FTP — точная настройка

- **Type**: FTP
- **Host**: IP-адрес или доменное имя FTP-сервера (например, IP NAS
  или внешнего FTP-хостинга)
- **Port**: `21` (стандартный control-порт FTP; менять не нужно, если
  сервер не настроен на нестандартный порт)
- **Username / Password**: учётные данные FTP-аккаунта
- **Base path**: папка на FTP-сервере, в которую писать файлы —
  например `/scans` или `/` (корень аккаунта), в зависимости от того,
  как настроен сам FTP-сервер

Важные ограничения текущей реализации:
- Поддерживается **только обычный FTP**, без шифрования (FTPS/SFTP не
  реализованы). Если сервер физически не в надёжной локальной сети —
  использовать вместо FTP вариант Synology NAS по HTTPS (раздел 4.3) или
  держать FTP-канал внутри VPN.
- Используется пассивный режим (passive mode) по умолчанию — это
  совместимо с большинством роутеров без доп. настройки на стороне
  клиента; но если FTP-сервер находится за NAT/файрволом и доступен
  снаружи локальной сети, на самом FTP-сервере должен быть настроен и
  открыт диапазон пассивных портов.
- Перед тем как заводить подключение в админке, стоит проверить
  доступ тем же логином/паролем через обычный FTP-клиент (FileZilla,
  WinSCP) — это сразу отделяет проблему сети/доступа от проблемы
  настройки в Doc Capture.

### 4.3 Synology NAS — точная настройка (через WebDAV)

Подключение к Synology идёт через встроенный в DSM сервис WebDAV —
отдельный FTP-сервер на NAS для этого не требуется.

**Шаг 1 — включить WebDAV на NAS:**
1. В DSM открыть **Панель управления → Службы файлов → WebDAV**.
2. Включить **HTTP** (порт по умолчанию `5005`) и/или **HTTPS**
   (порт по умолчанию `5006`, рекомендуется, если NAS доступен не
   только из локальной сети).
3. Применить.

**Шаг 2 — учётная запись и права:**
1. **Панель управления → Пользователи и группы** — создать (или
   выбрать существующего) пользователя DSM специально для Doc Capture.
2. Дать этому пользователю права на чтение/запись на нужную общую
   папку (Shared Folder) — именно в неё будут складываться файлы.

**Шаг 3 — узнать точный путь WebDAV:**
WebDAV на Synology показывает корень как список общих папок, на
которые у пользователя есть права — то есть путь обычно выглядит как
`/ИмяОбщейПапки/...`, **без** префикса `/volume1/`, который виден в
File Station. Точный путь может немного отличаться в зависимости от
версии DSM, поэтому **перед настройкой в админке стоит проверить
доступ сторонним WebDAV-клиентом**: на Windows — "Подключить сетевой
диск" → "Подключение к сайту" с адресом
`http://<IP-NAS>:5005/ИмяОбщейПапки`, либо приложением вроде Cyberduck.
Если путь открылся и видно содержимое папки — этот же путь и нужно
указать как **Base path** в Doc Capture.

**Шаг 4 — создать подключение в Doc Capture:**
- **Type**: Synology NAS (WebDAV)
- **Host**: локальный IP NAS (например, `192.168.1.20`) или его
  DDNS-адрес (например, `mynas.synology.me`), если сервер обращается
  к NAS не из той же локальной сети
- **Port**: `5005` для HTTP или `5006` для HTTPS
- **HTTPS**-чекбокс: включить, если используется порт `5006`/HTTPS —
  без этого галочка по умолчанию выключена, и подключение пойдёт по
  HTTP даже если указан порт 5006
- **Username / Password**: учётные данные DSM-пользователя из Шага 2
- **Base path**: путь общей папки, подтверждённый в Шаге 3, например
  `/DocCaptureScans`

## 5. Привязка хранилища к пользователю (Storage routing)

Подключение из раздела 4 само по себе ничего не делает — нужно явно
указать, в какое подключение конкретный пользователь должен сохранять
документы и в какое — фото (это может быть одно и то же подключение).

1. Открыть **Storage routing**.
2. Выбрать пользователя из списка.
3. **Document storage connection** / **Photo storage connection** —
   выбрать нужные подключения (можно одинаковые).
4. **Document/Photo subfolder pattern** — те же плейсхолдеры, что в
   разделе 3, определяют структуру подкаталогов внутри Base path
   подключения. Например, шаблон `{date}/{place}` для подключения с
   Base path `/DocCaptureScans` положит файл в
   `/DocCaptureScans/2026-06-17/TelAviv/имя_файла.pdf`.
5. **Save routing**.

Без этого шага загрузка из мобильного приложения для данного
пользователя будет завершаться ошибкой.

## 6. Журнал файлов (File log)

Список всех обработанных и сохранённых файлов: сгенерированное имя,
тип (документ/фото), место, итоговый путь, дата загрузки. Фильтр по
типу — в выпадающем списке наверху. Это журнал только для просмотра,
без возможности удаления файлов из интерфейса (файлы удаляются
напрямую в хранилище, если нужно).

## 7. Типичные проблемы

- **"No storage connection configured for this user"** при загрузке —
  для пользователя не выполнен раздел 5.
- **FTP/NAS-подключение не сохраняется или загруженные файлы не
  появляются** — проверить Host/Port/учётные данные тем же сторонним
  клиентом (FileZilla для FTP, WebDAV-клиент для Synology) в обход
  Doc Capture, чтобы исключить проблему сети/прав на стороне самого
  хранилища.
- **Synology: подключение "виснет" или ошибка 401/403** — чаще всего
  не включён нужный протокол (HTTP/HTTPS) в Шаге 1, либо у
  DSM-пользователя нет прав на конкретную общую папку (Шаг 2), либо
  не совпадает порт с выбранным HTTP/HTTPS-чекбоксом.
- **Файлы появляются, но не в той вложенной папке** — проверить
  subfolder pattern в разделе Storage routing и Base path самого
  подключения в разделе 4: итоговый путь — это всегда Base path +
  subfolder pattern + имя файла.
