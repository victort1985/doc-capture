# Doc Capture — обзор проекта

Кроссплатформенное решение для захвата, обработки и хранения документов и фото.

- **Сервер** (Windows/Linux): авторизация, хранение файлов (локально/FTP/Synology NAS), админка, шаблоны именования.
- **Клиент** (Android/iOS): авторизация, выбор места/типа документа, камера/файловый менеджер, пакетная загрузка, автообработка.

## Стек

- Backend: Node.js / NestJS, PostgreSQL (TypeORM), JWT-авторизация.
- Admin panel: React + Vite, без отдельного фреймворка состояния (useState/useEffect).
- Mobile: Flutter (Android/iOS из одной кодовой базы), Provider для состояния.

## Локализация мобильного клиента

Приложение по умолчанию полностью на иврите (he). При выборе другого языка в настройках
или на экране входа весь интерфейс переключается на выбранный язык. Поддерживаемые языки:
иврит (he, RTL), английский (en), русский (ru). Реализовано через Flutter ARB-файлы
(`lib/l10n/app_he.arb`, `app_en.arb`, `app_ru.arb`) — иврит является template-arb-file,
направление текста (RTL/LTR) Flutter определяет автоматически по выбранной locale.
